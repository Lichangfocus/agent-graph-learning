# 可行性实验：AI 构建知识关联的质量到底够不够？

**问题**：整合知识和先修关联是严谨、需要校验的工作，AI 实时构图能达到什么质量？

**方法**（2026-07 实验）：以专家人工构建的 [Marble os-taxonomy](https://github.com/withmarbleapp/os-taxonomy) 为标准答案，从数学·分数领域取 20 个知识点的核心簇（内含 27 条专家标注先修边，22 硬 5 软）。两个互相独立的 AI（分别扮演课程设计师、学习科学家）只拿到打乱且匿名的知识点列表（名称/描述/年龄段），盲推先修边，再与专家答案对比。

## 结果

| 指标 | 构图 A | 构图 B |
|---|---|---|
| 严格查准率（方向完全一致） | 62% | 53% |
| 查全率 | 74% | 70% |
| **硬先修边查全率** | **82%** | **77%** |
| 方向反转错误 | 1/32 | 2/36 |
| 产出图为合法 DAG | ✓ | ✓ |

## 错误性质分解（关键发现）

- "多提的边"里只有 1-2 条是方向反转（最危险的错误类型）；1-3 条是传递冗余（逻辑不错、只是不精简，对解锁逻辑无害）
- 其余"专家没标的新边"，两个 AI 的独立产出高度重合（10 条中 8 条相同）；抽 3 条给第三个对抗审查 AI 盲判，全部被判教学上成立——多半不是 AI 的错，而是专家图谱偏精简
- 两个 AI 共同"犯错"的一条边（词汇 vs 动手的先后），对抗审查 AI 也站在 AI 一边反对专家标注，且从年龄段看 AI 方向更符合"先动手再学术语"——**专家标注本身也不是无争议的，"严谨"的真实上限是专家间一致性**

## 结论与工程转化

1. 载荷最重的结构（硬先修）单次生成即可找回约 80%，交叉验证更高；方向错误率 3-6% 且集中在专家也会争论的边上
2. 校验必须自动化且已实现：本仓库 `scripts/validate_graph.py` 做环检测、年龄反向警告（抓方向反转）、传递冗余、quiz 完整性；SKILL.md 要求校验 PASS 才能出货，严肃主题另派独立 agent 盲审
3. 产品设计吸收残余误差：只有硬边锁内容、软边只影响顺序；掌握确认环节兜底
4. 分层策略：高利害主干用人工审核的预置图谱，兴趣探索用实时生成——次优边的代价本来就低

**实验局限**：单领域、20 节点；生成器与校验器同模型家族（生产环境建议跨家族互查）；测的是结构关联，事实性内容另有联网核实这条独立质检线。
