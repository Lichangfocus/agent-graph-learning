# learn-map：主题 → 知识地图学习站点

把一个学习主题变成"先修关系驱动"的探险地图站点：知识点是图上的星星，点亮先修才能解锁下一个，每个知识点是 3-5 屏图文卡 + 掌握确认 quiz。站点是单文件 HTML，进度存 localStorage，可导出/导入。

**核心理念——双层知识结构**：每个知识点分两层。①**原始知识层（canon）**：教科书/课标/论文里的标准术语、正式表述和真实出处——学习的"锚"，保证不歪曲、可对照课内；②**通俗重构层（screens）**：把 canon 重构成孩子能懂、愉快可玩的图文卡。学习者随时可以点"课本上怎么说"在两层之间切换。同时，**教学顺序被显式重构**：不按教科书章节顺序，而按理解关系（甚至跨学科）排序，且每张图必须用 `reorder` 字段声明自己和教科书顺序的差别及理由。

## 流程（严格按顺序执行）

### 第 0 步：对焦

如果用户没有说明，用 AskUserQuestion 问清楚（一次问完，最多 3 问）：
1. **学习者年龄/水平**（如 6-8 岁 / 8-10 岁 / 10-12 岁 / 成人入门）
2. **切入角度**：这个主题里最想搞明白什么（给 3 个具体选项）
3. **已有基础**（可选）：学过什么相关的东西

### 第 1 步：构建图谱（先落原始知识层）

生成 **12-20 个知识点** 和它们之间的先修边，写入工作目录 `学习地图/<主题slug>/site.json`。

数据 schema（模板依赖这个结构，不要改字段名）：

```json
{
  "id": "sound-journey",            // 站点唯一 id（英文 slug，决定 localStorage key）
  "title": "声音之旅 🎵",
  "tag": "主题探险线 · SOUND",       // 顶部小字
  "about": "写给家长看的一段话，说明数据从哪来、进度怎么存",
  "reorder": "必填：一段话声明这张图的顺序与教科书顺序有何不同、为什么这样更易理解（学习档案中展示）",
  "nodes": [{
    "id": "vibration",              // 英文短 slug
    "icon": "🎸",                   // 一个 emoji
    "name": "振动与声音",            // ≤7 个字，好奇心驱动的命名
    "domain": "科学",               // 领域名，同领域自动同色（≤5 个领域）
    "age": "6-9岁",                 // 展示用
    "ageStart": 6,                  // 校验用（数字）
    "desc": "一句话描述这个知识点",
    "canon": {                      // 原始知识层（必填）
      "term": "标准术语（含英文原名，如适用）",
      "formal": "教科书/课标式正式表述，保持严谨、不做通俗化；引用原文时中英对照",
      "source": "真实出处：课标条目代码（如 NGSS 1-PS4-1 / CCSS 3.MD.B.3 / UK NC Y4.Sci.Snd.1）、教材章节或论文",
      "textbook": "它在教科书体系中的位置（年级/单元），用于顺序对比"
    },
    "evidence": ["2-3 条可观察的掌握证据，'能说出/能演示/能解释…'"],
    "screens": [
      { "h": "小标题", "body": "<p>HTML 内容…</p>" },
      { "quiz": { "q": "问题", "opts": [{"t": "选项", "ok": true}, {"t": "干扰项"}, {"t": "干扰项"}], "fb": "答对后的解释", "hint": "答错时的提示" } }
    ]
  }],
  "edges": [
    { "from": "先修id", "to": "后继id", "type": "hard" }   // hard=必须先学(锁解锁) / soft=学过更好 / cross=跨领域
  ]
}
```

构图规则：
- **canon 先行**：先为每个节点检索/核实原始知识层（term/formal/source/textbook），再设计边和通俗内容。出处必须真实——课标代码、教材章节、论文引用不确定的，先 WebSearch 核实，宁缺勿造
- **reorder 必填**：明确说出这张图哪里偏离了教科书顺序、为什么（这是本方法论的灵魂，不是可选项）
- **有向无环**；每个节点典型 1-3 条直接硬先修；**不要传递闭包**（有 A→B→C 就别再加 A→C）
- 至少 1 个无硬先修的起点；硬边上先修的 ageStart ≤ 后继的 ageStart
- 只有 hard 边参与解锁；soft/cross 边只画在地图上传达关联
- 如果主题天然跨领域，放 1-2 个跨领域节点（比如"找规律"这类通用能力），用 cross 边连接——跨学科看到本质是这个产品的灵魂
- 布局不用管：模板会按先修层级自动排版

### 第 2 步：校验（必做，不通过不许继续）

```bash
python3 <skill目录>/scripts/validate_graph.py 学习地图/<slug>/site.json
```

- ERROR 必须修复后重跑；WARN 逐条判断（年龄反向警告优先检查是不是边的方向写反了——这是最危险的错误类型）
- 图谱较大或主题严肃（学科知识）时，额外派一个独立 subagent 只看节点列表盲审边的方向与合理性，交叉比对后再定稿

### 第 3 步：写图文卡内容（易读易学习规范）

每个节点 3-5 屏（最后 1-2 屏必须是 quiz）。逐条遵守：

1. **一屏一个念头**。每屏 ≤3 段短句，一屏只推进一个认知步骤：是什么 → 生活里找一找 → 动手试一试 → 我来回答
2. **首屏是钩子不是定义**。从学习者的身体经验或生活场景切入（"把手放在喉咙上说句话——感觉到抖了吗？"），定义放在体验之后
3. **用已学的东西打比方**。行文里主动引用该节点的先修概念（"还记得你学过的 X 吗"），把图谱结构变成叙事结构
4. **具象优先**。每个抽象概念至少配一样：内联 SVG 小图（可加简单 animate）、`.try-list` 动手实验清单（用家里就有的东西）、或 `.stage` 包裹的示意图。模板已有样式类：`.stage`（白底展示区）、`.try-list`/`.try-item`（带序号实验步骤）、`<b>`（关键词高亮）
5. **语言分级**。6-8 岁：短句、口语、每屏 ≤60 字；9-12 岁：可用学名但首次出现必须解释；成人：正常行文但保持"每屏一个念头"
6. **quiz 三原则**：题目测 evidence 里写的行为，不测记忆；3 个选项，错误项必须"有意义地错"（对应真实的常见误解，其中一个可以幽默但不能全是搞笑）；fb 解释为什么对，hint 给方向不给答案
7. **屏幕内容必须覆盖该节点每一条 evidence**——evidence 是内容的验收标准，不是装饰
8. **双层一致性**：通俗层是 canon 的重构，不是替代——不得与 canon 冲突或歪曲原意；关键术语首次出现时用 canon 的标准叫法（可以随后起昵称）；卡片上的"课本上怎么说"按钮就是两层之间的桥，写通俗层时想着"孩子点开原文时应该能认出这是同一件事"

### 第 4 步：组装站点

1. 复制 `<skill目录>/assets/template.html` 到 `学习地图/<slug>/index.html`
2. 把文件中的 `/*__DATA__*/null` 整体替换为 site.json 的内容（**只替换第一处**，用 Python `str.replace(..., 1)` 而不是全局替换）
3. 用浏览器打开验证：起点节点可点开、翻屏正常、quiz 答对后能"点亮"、刷新页面进度还在
4. 告诉用户站点路径，说明：进度存在浏览器 localStorage（按站点 id 隔离）；换设备用"学习档案 → 导出/导入进度"迁移；卡片里的"问 AI"按钮会复制一句带上下文的提问（含年龄、当前知识点、已学内容），可粘贴给任何 AI 追问

## 多章教材模式（大主题）

主题超过 ~20 个知识点时（比如"AI 大模型训练全链路"这类完整教材），不要塞进一张图，按以下模式分章：

1. **切章**：按理解阶段切成 4-6 章，每章 12-20 个节点、独立成图（独立 site.json + index.html，放在 `maps/<主题>/ch1-xxx/`、`ch2-xxx/`…）。章内边照常；跨章依赖不连边，写进后一章的 about（"建议先完成第 N 章"）
2. **骨架先行，内容并行**：先由主控者设计好全部章节的节点/边/canon 出处（保证跨章连贯、术语一致），再把每章内容交给并行 subagent 生成——每个 agent 拿到完整图结构（不增不减不改方向）+ 内容规范 + 真实出处清单，只负责写内容和跑校验
3. **总入口**：用 `assets/hub-template.html`（占位符 `/*__HUB_DATA__*/null`，同样只替换第一处）生成 `maps/<主题>/index.html`。HUB 数据格式：
   `{"title","tag","about","foot","chapters":[{"dir":"ch1-xxx","id":"<该章site.json的id>","icon","title","desc","total":<节点数>}]}`
   入口页会自动读各章的 localStorage 进度（同源），汇总总进度、标记"从这里继续"
4. **主控者终检**：逐章跑校验器 + 抽查内容（术语跨章一致性、canon 出处真实性、quiz 可答性），再组装发布

站点 id 命名：`<主题>-<章号>-<slug>`（如 ai-1-foundations），id 决定 localStorage key，发布后不要改。

## 质量红线

- 事实性内容不确定的，生成前先 WebSearch 核实（尤其科学数据、历史年代）
- 不为了凑数造节点；12 个高质量节点好过 20 个注水节点
- 主题涉及英美课标学科（数学/科学/英语）时，如果本地有 Marble os-taxonomy 数据（检查 scratchpad 或用户目录下的 os-taxonomy/），优先检索复用其真实节点与边，只生成缺口
